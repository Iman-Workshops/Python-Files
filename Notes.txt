# یادداشت‌های کارگاه کار با فایل‌ها در پایتون

این یادداشت‌ها برای تدریس یا خودخوانی طراحی شده‌اند. هدف این است که دانشجو فقط چند دستور حفظ نکند، بلکه بداند فایل چیست، چرا باید بسته شود، کدگذاری چه اثری دارد و چگونه از داده خام به گزارش قابل استفاده برسد.

## 1. هدف کارگاه

در پایان مسیر، دانشجو باید بتواند:

- فایل متنی را با حالت‌های اصلی باز کند.
- متن فارسی را با کدگذاری درست بنویسد و بخواند.
- فایل‌های بزرگ را بدون مصرف زیاد حافظه خط به خط بخواند.
- مسیر فایل را با `pathlib.Path` بسازد.
- داده JSON و CSV را بخواند و بنویسد.
- لاگ برنامه را در فایل نگه دارد.
- خطاهای رایج فایل را مدیریت کند.
- تفاوت فایل متنی و فایل باینری را توضیح دهد.
- یک CSV را بخواند، داده را فیلتر کند و گزارش JSON بسازد.

## 2. فایل چیست؟

برنامه هنگام اجرا داده را در حافظه نگه می‌دارد. حافظه موقتی است. اگر داده باید بعد از پایان برنامه باقی بماند، باید روی دیسک ذخیره شود. فایل یکی از ساده‌ترین راه‌های ذخیره داده است.

نمونه‌های رایج فایل در پروژه‌های پایتون:

- فایل متنی برای یادداشت، گزارش و خروجی قابل خواندن
- فایل JSON برای تنظیمات یا داده ساخت‌یافته
- فایل CSV برای داده جدولی
- فایل log برای ثبت مسیر اجرای برنامه
- فایل باینری برای تصویر، صدا، فایل فشرده یا داده خام

## 3. حالت‌های باز کردن فایل

تابع `open` یک مسیر و یک حالت می‌گیرد. حالت مشخص می‌کند فایل چگونه باز شود.

- حالت `r`: خواندن. فایل باید وجود داشته باشد.
- حالت `w`: نوشتن از ابتدا. اگر فایل وجود داشته باشد پاک می‌شود.
- حالت `a`: اضافه کردن به انتهای فایل. اگر فایل وجود نداشته باشد ساخته می‌شود.
- حالت `x`: ساخت فایل تازه. اگر فایل وجود داشته باشد خطا می‌دهد.
- حالت `r+`: خواندن و نوشتن روی فایل موجود.

نکته مهم درباره `r+`: فایل ساخته نمی‌شود و نشانگر فایل اهمیت دارد. اگر برنامه اول فایل را بخواند، نشانگر جلو می‌رود. برای نوشتن در جای مشخص باید از `seek` استفاده شود.

نمونه:

```python
with open("sample.txt", "r+", encoding="utf-8") as file:
    first_line = file.readline()
    file.seek(0, 2)
    file.write("خط تازه\n")
```

دستور `seek(0, 2)` نشانگر فایل را به انتهای فایل می‌برد.

## 4. چرا with مهم است؟

باز کردن فایل یعنی برنامه از سیستم‌عامل یک منبع می‌گیرد. اگر فایل بسته نشود، چند مشکل ممکن است رخ دهد:

- همه داده‌ها کامل روی دیسک نوشته نشوند.
- فایل برای برنامه‌های دیگر قفل بماند.
- در برنامه‌های طولانی، تعداد فایل‌های باز زیاد شود.

عبارت `with` یک context manager است. این روش فایل را بعد از پایان بلوک می‌بندد، حتی اگر خطا رخ دهد.

```python
with open("sample.txt", "w", encoding="utf-8") as file:
    file.write("متن نمونه\n")
```

اگر به هر دلیل از `with` استفاده نشود، باید بستن فایل در `finally` انجام شود.

```python
file = open("sample.txt", "r", encoding="utf-8")
try:
    content = file.read()
finally:
    file.close()
```

## 5. خواندن فایل متنی

روش‌های رایج خواندن:

- روش `read`: کل فایل را یک رشته می‌کند. برای فایل کوچک خوب است.
- روش `readline`: یک خط می‌خواند. برای کنترل دقیق مناسب است.
- روش `readlines`: همه خط‌ها را در لیست می‌گذارد. برای فایل بزرگ حافظه زیادی مصرف می‌کند.
- حلقه روی فایل: هر بار یک خط می‌خواند و برای فایل بزرگ مناسب‌تر است.

برای فایل بزرگ، این روش ترجیح دارد:

```python
with open("large.txt", "r", encoding="utf-8") as file:
    for line in file:
        process(line)
```

در این حالت لازم نیست همه فایل یکجا وارد حافظه شود.

## 6. نوشتن و اضافه کردن متن

حالت `w` برای ساخت خروجی تازه مناسب است. هر بار اجرای برنامه، فایل قبلی جایگزین می‌شود.

حالت `a` برای اضافه کردن رخداد، یادداشت یا لاگ دستی به انتهای فایل مناسب است.

```python
with open("notes.txt", "a", encoding="utf-8") as file:
    file.write("خط تازه\n")
```

در کلاس، یک تمرین خوب این است که دانشجو اول فایل را با `w` اجرا کند، سپس با `a` و خروجی را در پوشه `workshop_output` ببیند.

## 7. کدگذاری UTF-8 و متن فارسی

فایل متنی در واقع مجموعه‌ای از بایت‌ها است. کدگذاری مشخص می‌کند هر حرف چگونه به بایت تبدیل شود. برای فارسی، `utf-8` انتخاب رایج و قابل حمل است.

اگر کدگذاری نوشته نشود، پایتون از پیش‌فرض سیستم استفاده می‌کند. این پیش‌فرض در همه سیستم‌ها یکسان نیست. در بعضی محیط‌های ویندوز، خواندن یا نوشتن فارسی بدون کدگذاری روشن به خطا می‌رسد.

همیشه برای متن فارسی بنویسید:

```python
with open("persian.txt", "w", encoding="utf-8") as file:
    file.write("سلام پایتون\n")
```

### مفهوم BOM

عبارت BOM به چند بایت در ابتدای بعضی فایل‌های متنی اشاره دارد. در UTF-8 معمولا لازم نیست، اما بعضی برنامه‌ها آن را می‌نویسند. اگر فایل دارای BOM مشکل ساخت، می‌توان از `utf-8-sig` استفاده کرد.

```python
text = Path("file.txt").read_text(encoding="utf-8-sig")
```

### خطای decode

اگر فایل با یک کدگذاری نوشته شده باشد و با کدگذاری دیگری خوانده شود، خطایی مثل `UnicodeDecodeError` رخ می‌دهد. پیام خطا معمولا نشان می‌دهد پایتون در کدام بایت نتوانسته متن را تفسیر کند.

## 8. مسیرها با pathlib

در ویندوز مسیرها با `\` دیده می‌شوند. در مک و لینوکس مسیرها با `/` دیده می‌شوند. چسباندن رشته‌ها برای ساخت مسیر، کد را شکننده می‌کند.

کلاس `pathlib.Path` مسیر را به شکل قابل حمل می‌سازد:

```python
from pathlib import Path

base_dir = Path(__file__).resolve().parent
output_dir = base_dir / "workshop_output"
output_dir.mkdir(exist_ok=True)
```

کارهای مهم با `Path`:

- متد `path.exists()` برای بررسی وجود مسیر
- متد `path.mkdir()` برای ساخت پوشه
- متدهای `path.read_text()` و `path.write_text()` برای متن
- متدهای `path.read_bytes()` و `path.write_bytes()` برای باینری
- متد `path.glob("*.txt")` برای پیدا کردن فایل‌ها با الگو

## 9. قالب JSON

قالب JSON برای داده ساخت‌یافته مناسب است. دیکشنری، لیست، رشته، عدد، مقدار بولی و `None` در پایتون به JSON نزدیک هستند.

نمونه داده تو در تو:

```python
profile = {
    "name": "سارا",
    "skills": ["Python", "JSON"],
    "scores": {"files": 18.5, "csv": 17.25},
}
```

نوشتن:

```python
with open("profile.json", "w", encoding="utf-8") as file:
    json.dump(profile, file, ensure_ascii=False, indent=2)
```

خواندن:

```python
with open("profile.json", "r", encoding="utf-8") as file:
    profile = json.load(file)
```

نکته‌ها:

- گزینه `ensure_ascii=False` متن فارسی را خوانا نگه می‌دارد.
- گزینه `indent=2` خروجی را مرتب می‌کند.
- تابع `json.load` با فایل کار می‌کند.
- تابع `json.loads` با رشته JSON کار می‌کند.
- اگر فایل JSON خراب باشد، `json.JSONDecodeError` رخ می‌دهد.

## 10. قالب CSV

قالب CSV برای داده جدولی مناسب است. هر ردیف یک رکورد و هر ستون یک ویژگی است. اگر فایل سرستون دارد، کار با `DictReader` و `DictWriter` روشن‌تر است.

نوشتن CSV:

```python
with open("people.csv", "w", encoding="utf-8", newline="") as file:
    writer = csv.DictWriter(file, fieldnames=["name", "age", "city"])
    writer.writeheader()
    writer.writerow({"name": "رضا", "age": 28, "city": "یزد"})
```

خواندن CSV:

```python
with open("people.csv", "r", encoding="utf-8", newline="") as file:
    reader = csv.DictReader(file)
    for row in reader:
        print(row["name"])
```

### گزینه newline

در فایل CSV، بهتر است هنگام باز کردن فایل از `newline=""` استفاده شود. این کار جلوی خط خالی ناخواسته در بعضی محیط‌ها را می‌گیرد.

### جداکننده و نقل‌قول

همه CSVها با ویرگول جدا نمی‌شوند. بعضی فایل‌ها از `;` یا `\t` استفاده می‌کنند.

```python
reader = csv.reader(file, delimiter=";")
```

اگر مقدار یک ستون خودش ویرگول یا کوتیشن داشته باشد، ماژول `csv` آن را با قواعد نقل‌قول درست ذخیره و بازیابی می‌کند.

## 11. لاگ

دستور `print` برای نمایش سریع مناسب است، اما برای ثبت مسیر اجرای برنامه بهتر است از `logging` استفاده شود. لاگ می‌تواند زمان، سطح و پیام را در فایل نگه دارد.

سطح‌ها:

- سطح `DEBUG`: جزئیات عیب‌یابی
- سطح `INFO`: رخداد عادی
- سطح `WARNING`: هشدار
- سطح `ERROR`: خطا

قالب رایج:

```python
%(asctime)s %(levelname)s %(message)s
```

برای پروژه پایانی، لاگ کمک می‌کند مشخص شود چند ردیف خوانده شده، چند ردیف رد شده و گزارش کجا ساخته شده است.

## 12. مدیریت خطا

خطاهای رایج:

- خطای `FileNotFoundError`: فایل یا پوشه وجود ندارد.
- خطای `PermissionError`: برنامه اجازه خواندن یا نوشتن ندارد.
- خطای `UnicodeDecodeError`: فایل با کدگذاری مورد انتظار خوانده نمی‌شود.
- خطای `json.JSONDecodeError`: JSON خراب است.
- خطای `ValueError`: تبدیل رشته به عدد ممکن نیست.

نمونه:

```python
try:
    content = path.read_text(encoding="utf-8")
except FileNotFoundError:
    print("فایل پیدا نشد")
except PermissionError:
    print("اجازه خواندن فایل وجود ندارد")
```

بخش `finally` برای کاری مناسب است که باید در هر حالت انجام شود، مثل بستن فایل.

## 13. فایل متنی و فایل باینری

فایل متنی با `str` کار می‌کند و نیاز به کدگذاری دارد. فایل باینری با `bytes` کار می‌کند و کدگذاری متن در آن دخالت ندارد.

متنی:

```python
Path("note.txt").write_text("سلام", encoding="utf-8")
```

باینری:

```python
Path("data.bin").write_bytes(bytes([0, 1, 2, 3]))
```

اگر فایل تصویر، صدا یا فایل فشرده باشد، معمولا باید به صورت باینری خوانده شود.

## 14. پروژه پایانی

پروژه پایانی چند مفهوم را کنار هم می‌گذارد:

1. ساخت یا دریافت فایل CSV
2. خواندن با `DictReader`
3. تبدیل نمره از رشته به عدد
4. رد کردن ردیف نامعتبر با پیام لاگ
5. فیلتر کردن دانشجوهای قبول شده
6. محاسبه میانگین
7. ذخیره گزارش JSON با `ensure_ascii=False`
8. ثبت مسیر اجرا در فایل لاگ

این پروژه نشان می‌دهد کار با فایل فقط خواندن و نوشتن نیست. برنامه باید داده خام را به خروجی قابل اعتماد تبدیل کند و مسیر کار خود را هم ثبت کند.

## 15. نوشتن امن فایل

وقتی برنامه فایل مهمی را بازنویسی می‌کند، یک خطر وجود دارد: اگر برنامه وسط نوشتن متوقف شود، فایل ممکن است ناقص بماند. روش بهتر این است که ابتدا خروجی در فایل موقت نوشته شود. وقتی نوشتن کامل شد، فایل موقت جای فایل اصلی را بگیرد.

مراحل کلی:

1. اگر فایل قبلی مهم است، نسخه پشتیبان ساخته شود.
2. داده در یک فایل موقت در همان پوشه نوشته شود.
3. فایل موقت با `replace` جای فایل اصلی را بگیرد.

این روش احتمال فایل نیمه‌کاره را کم می‌کند. نمونه 14 این الگو را با JSON نشان می‌دهد.

## 16. پیمایش پوشه و گزارش فایل‌ها

در بسیاری از پروژه‌ها برنامه باید بداند چه فایل‌هایی در یک پوشه وجود دارد. `pathlib` برای این کار چند ابزار مهم دارد:

- متد `iterdir` فقط محتوای مستقیم یک پوشه را می‌دهد.
- متد `glob` فایل‌ها را با الگو پیدا می‌کند.
- متد `rglob` در زیرپوشه‌ها هم جست‌وجو می‌کند.
- متد `stat` اندازه و اطلاعات زمانی فایل را می‌دهد.

برای ساخت گزارش از پوشه، بهتر است مسیر فایل‌ها نسبت به پوشه اصلی ذخیره شود. مسیر مطلق روی سیستم‌های مختلف فرق می‌کند، اما مسیر نسبی قابل انتقال‌تر است.

## 17. فایل zip

فایل zip چند فایل را در یک آرشیو نگه می‌دارد. پایتون با ماژول `zipfile` می‌تواند فایل zip بسازد، فهرست داخل آن را بخواند و محتوا را استخراج کند.

کارهای اصلی:

- ساخت آرشیو با `ZipFile(..., "w")`
- اضافه کردن فایل با `archive.write`
- خواندن فهرست با `namelist`
- استخراج با `extractall`

هنگام ساخت آرشیو، نام داخل zip با `arcname` کنترل می‌شود. اگر `arcname` مشخص نشود، ممکن است مسیر کامل فایل وارد zip شود.

## 18. پردازش جریانی داده

برای فایل CSV کوچک می‌توان همه ردیف‌ها را در لیست نگه داشت. برای فایل بزرگ، این کار حافظه زیادی مصرف می‌کند. روش بهتر این است که ردیف‌ها یکی یکی خوانده شوند و همان لحظه پردازش شوند.

تابع generator برای این کار مناسب است:

```python
def iter_rows(path):
    with path.open("r", encoding="utf-8", newline="") as file:
        reader = csv.DictReader(file)
        for row in reader:
            yield row
```

در این روش، برنامه لازم نیست همه داده را هم‌زمان در حافظه نگه دارد.

## 19. فایل تنظیمات INI

برای تنظیمات ساده برنامه، فایل INI خواناتر از JSON است. ماژول `configparser` بخش‌ها و کلیدها را می‌خواند.

نمونه فایل:

```ini
[app]
language = fa
autosave = yes
theme = light
```

مزیت INI این است که برای انسان خواناست. محدودیت آن این است که برای داده تو در تو مثل JSON مناسب نیست.
